# GDPR Expert 사용 가이드

[English](../en/HOW-TO-USE.md) | [한국어](./HOW-TO-USE.md)

> 이 가이드는 **비개발자**를 위해 작성되었습니다. Python, Git, API를 몰라도 괜찮습니다. 질문을 타이핑할 수 있으면 이 도구를 사용할 수 있습니다.

---

## 필요한 것 (최초 1회 설정)

| 항목 | 이유 | 설치 방법 |
|------|------|---------|
| **Claude Code** | 에이전트를 실행하는 앱 | [다운로드](https://claude.ai/claude-code) — CLI, 데스크톱 앱, VS Code 확장 |
| **이 저장소** | 법령 Knowledge Base + 에이전트 포함 | 팀 리드 또는 IT에서 제공 |

이게 전부입니다. 데이터베이스, 서버, API 키 설정이 필요 없습니다 (법령 데이터를 직접 갱신하려는 경우 제외).

## 에이전트 시작하기

### 방법 A: 데스크톱 앱 / VS Code

1. Claude Code 실행
2. `GDPR-expert` 폴더 열기
3. 에이전트가 자동 활성화 — 법령 Knowledge Base에 접근 가능

### 방법 B: 터미널 (CLI)

```bash
cd GDPR-expert
claude --agent .claude/agents/gdpr-agent.md
```

## 질문하기

자연어로 질문하면 됩니다. **영어**와 **한국어** 모두 이해하며, 요청 시 주요 EU 언어로도 응답할 수 있습니다.

### 단순 조회

> "GDPR 제6조 보여줘"

> "제17조의 삭제권(잊힐 권리)에 대해 알려줘"

> "EDPB의 동의(consent) 가이드라인은?"

### 법률 분석

> "기존 B2B 고객에게 마케팅 이메일을 보내는 데 legitimate interest를 근거로 삼을 수 있나?"

> "제37-39조에 따른 DPO 선임 요건은?"

> "GDPR과 NIS2 Directive의 데이터 위반 통지 요건 비교해 줘"

### 교차참조

> "제6조(1)(f) legitimate interest 형량 테스트를 설명하는 Recital은?"

> "제17조 삭제권을 해석한 CJEU 판례는?"

> "쿠키 동의에 대해 ePrivacy Directive와 GDPR은 어떻게 상호작용해?"

### 미래 전망 분석

> "Digital Omnibus Package가 채택되면 위반 통지 규정이 어떻게 바뀌어?"

> "Digital Omnibus가 AI 학습에 대한 legitimate interest를 어떻게 변경하려고 해?"

## 법률의견서 요청하기 (DOCX)

채팅 답변이 아닌 정식 문서가 필요할 때 — 법률의견서를 요청하세요:

> "Schrems II 이후 직원 데이터를 미국으로 이전할 수 있는지에 대한 법률의견서를 작성해 줘. DOCX."

> "AI 기반 채용 도구에 대한 GDPR + AI Act 컴플라이언스 평가서 작성. 영어+한국어."

> "새 고객 프로파일링 기능에 대한 DPIA 분석. DOCX로."

에이전트가 수행하는 작업:
1. 전체 Knowledge Base 검색 (법령, 가이드라인, 판례)
2. 검증된 인용이 포함된 구조화된 의견서 초안 작성
3. 팩트체커로 모든 법적 인용 검증
4. 전문적인 DOCX 파일을 `output/opinions/`에 저장

### 이중언어 의견서

요청에 "영어와 한국어"를 추가하세요:

> "국제 데이터 이전에 대한 법률의견서 — DOCX, 영어+한국어 버전"

에이전트가 별도의 DOCX 파일 2개를 생성합니다. 한국어 버전에서 **법 원문 인용은 영어 원문 그대로** 유지되고 (EU법은 영어가 공식), 분석과 해설만 한국어로 작성됩니다.

## 결과물 이해하기

### 인용 태그

에이전트 출력의 모든 법적 인용에는 신뢰도 태그가 붙습니다:

| 태그 | 의미 | 신뢰할 수 있나? |
|------|------|---------------|
| `[VERIFIED] [Grade A]` | KB의 법령 원문과 정확히 매칭 | 높은 신뢰도 — 단, 중요한 포인트는 항상 재확인 |
| `[VERIFIED] [Grade B]` | KB의 DPA 집행결정 또는 법원 판결과 매칭 | 양호한 신뢰도 — 권위 있지만 2차 소스 |
| `[UNVERIFIED]` | 웹 검색으로 발견, 로컬 KB에 없음 | 의존 전 독립적으로 검증 필요 |
| `[INSUFFICIENT]` | 에이전트가 충분한 근거를 찾지 못함 | 에이전트가 솔직한 것 — 추측 말고 변호사 상담 |
| `[CONTRADICTED]` | 소스마다 다른 내용 | 양쪽 모두 제시됨 — 판단은 사용자 몫 |

### 소스 등급

| 등급 | 포함 내용 | 단독 근거로 사용 가능? |
|------|---------|---------------------|
| **A** | EU 법령 원문, EDPB 가이드라인, CJEU 판결 | 가능 |
| **B** | DPA 집행결정, 각국 법원 판결 | 가능 (Grade A 교차검증 권장) |
| **C** | 로펌 분석, 학술 논문 | 불가 — 참고용으로만 |
| **D** | 뉴스 기사, AI 요약 | 시스템에서 완전 제외 |

## 나만의 소스 추가하기

각국 DPA 결정, 내부 정책, 학술 논문 등을 에이전트에 추가하고 싶다면?

### Step 1: 파일 넣기

아무 파일(PDF, DOCX, HTML)을 `library/inbox/` 폴더에 넣습니다.

### Step 2: 에이전트에 알리기

> "inbox에 파일 넣었어, ingest 해줘"

또는:

> "/ingest"

### Step 3: 완료

에이전트가 자동으로:
- 파일을 Markdown으로 변환
- 신뢰도 자동 분류 (Grade A, B, C)
- 메타데이터 추출 (제목, 날짜, 키워드, 관련 GDPR 조문)
- 적절한 폴더에 배치
- 검색 인덱스 업데이트

새 소스가 바로 검색 가능하고, 향후 의견서에서 인용될 수 있습니다.

### 추가할 수 있는 자료 (예시)

| 자료 | 처리 결과 |
|------|---------|
| 독일 TTDSG (ePrivacy 국내 이행법) | Grade A로 분류, 법령 폴더에 배치 |
| 프랑스 CNIL 결정 PDF | Grade B로 분류, 집행결정 폴더에 배치 |
| 로펌 클라이언트 얼럿 (AI Act 관련) | Grade C로 분류, 참고자료 폴더에 배치 |
| 테크크런치 GDPR 과징금 기사 | Grade D로 거부 (경고 메시지 표시) |

## 좋은 결과를 위한 팁

### 구체적으로 질문하기

| 이렇게 말고 | 이렇게 |
|-----------|--------|
| "GDPR에 대해 알려줘" | "GDPR 제6조의 6가지 처리 적법 근거는?" |
| "이거 합법이야?" | "제28조에 따라 processor가 controller의 명시적 승인 없이 AI 학습에 데이터를 쓸 수 있어?" |
| "데이터 이전 어떻게 해?" | "Schrems II 판결 이후 미국으로 개인정보를 이전할 때 필요한 보호조치는?" |

### 반론도 요청하기

> "이 처리에 legitimate interest를 근거로 삼는 것에 반대하는 논거는 뭐가 있어?"

에이전트에 adversarial 모드가 내장되어 있어서 예외, 반대 판례, 대안적 해석을 찾지만, 명시적으로 요청하면 더 효과적입니다.

### 팩트체크 리포트 요청하기

중요한 의견서의 경우, 검증 결과를 확인하세요:

> "이 의견서의 팩트체크 리포트 보여줘"

어떤 인용이 검증되었는지, 문제가 있었는지, 전체 신뢰도 점수를 볼 수 있습니다.

## 이 도구가 하지 않는 것

- **법률 자문을 제공하지 않습니다.** 법률 소스를 더 빠르고 체계적으로 찾도록 도와주는 리서치 보조 도구입니다. 결과물은 반드시 법률 전문가가 검토해야 합니다.
- **귀하의 구체적인 계약이나 내부 정책을 알지 못합니다** — ingest 시스템으로 추가하지 않는 한.
- **자동으로 업데이트되지 않습니다.** 법령 데이터는 특정 시점의 스냅샷입니다. 주기적으로 갱신 스크립트를 실행하거나 ingest로 새 소스를 추가하세요.
- **EU Directive의 각국 이행법을 기본 포함하지 않습니다.** 예를 들어 ePrivacy Directive는 각 EU 회원국마다 다르게 전환되어 있습니다. ingest로 해당 국가의 이행법을 추가하세요.

## 도움이 필요할 때

- **뭔가 이상해 보인다면?** 에이전트에게: "제X조 인용을 검증해줘"
- **다른 포맷이 필요하다면?** "마크다운 요약으로 줘" 또는 "DOCX로 만들어줘"
- **에이전트가 막힌 것 같다면?** 질문을 다시 표현하거나 작은 부분으로 나눠서 시도
- **기술적 문제라면?** 팀 리드에게 문의하거나 [GitHub 저장소](https://github.com/kipeum86/GDPR-expert) 확인

---

> **기억하세요:** 이것은 자동 조종이 아니라 파워 도구입니다. 법률 리서치를 극적으로 빠르고 체계적으로 만들어주지만, 최종 판단은 항상 자격을 갖춘 사람의 몫입니다.
